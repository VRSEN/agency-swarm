## Before You Begin

- An Agencii account
- A GitHub account
- At least one provider API key (for example OpenAI or Anthropic)

## Deploy OpenClaw On Agencii

<Steps>
  <Step title="Create your repository">
    Open [agency-ai-solutions/openclaw-starter-template](https://github.com/agency-ai-solutions/openclaw-starter-template).

    Click **Use this template** and create your own repository.

    ![Use this template dropdown](/images/getting-started/github-use-this-template-dropdown.png)
  </Step>

  <Step title="Connect the repository in Agencii">
    In Agencii, connect your GitHub account and select the repository you created from the template.
  </Step>

  <Step title="Configure onboarding form (Marketplace)">
    If you plan to publish this template in Marketplace, add `onboarding_tool.py` in your repository root and define the onboarding fields.
    This is the form users fill before deployment.

    Use [Marketplace Onboarding](/platform/marketplace/onboarding) as the implementation guide.
  </Step>

  <Step title="Add provider keys in Agencii">
    Add provider API keys in the Agencii key modal.
    If keys are missing, OpenClaw cannot call model providers.

    Common keys:
    - `OPENAI_API_KEY`
    - `ANTHROPIC_API_KEY`
  </Step>

  <Step title="Deploy">
    Start deployment and wait for the build to complete.
    Agencii builds and runs the repository automatically.
  </Step>

  <Step title="Run your first chat">
    Open your deployed agency chat and send a simple test prompt.

    Example: `What can you automate for me today?`
  </Step>
</Steps>

## What You Get After Deploy

- A deployable OpenClaw-based agency on Agencii
- Your normal Agencii chat flow keeps working
- Persistent OpenClaw files under `/mnt/openclaw`
- Default OpenClaw model id: `openclaw:main`

## Keys And Security

Use BYOK in the Agencii key modal. Do not collect raw API keys in onboarding forms.

<Warning>
This setup is intended as your personal OpenClaw instance. Do not share one instance across untrusted users.
Provider API keys you add in Agencii are available inside the running sandbox environment so OpenClaw can call model providers.
Review [OpenClaw Security](https://docs.openclaw.ai/gateway/security) before opening wider access.
</Warning>

## If Something Is Not Working

- Agent does not respond:
  - re-check provider keys in Agencii key modal, then try again
- Deployment succeeds but responses fail:
  - open deployment logs in Agencii, then redeploy
- Data does not persist between restarts:
  - check that `/mnt` is mounted and writable in your runtime

<Accordion title="Advanced: technical details" defaultOpen={false}>

### OpenClaw model defaults

- Stable model id in agency config: `openclaw:main`
- `openclaw:main` maps to `OPENCLAW_PROVIDER_MODEL`
- Default provider model: `OPENCLAW_PROVIDER_MODEL=openai/gpt-5-mini`

### OpenClaw proxy endpoints

- `POST /openclaw/v1/responses`
- `GET /openclaw/health`

### Agency Swarm endpoints (unchanged)

- `POST /{agency_name}/get_response_stream`
- `POST /{agency_name}/cancel_response_stream`

### Stream and cancel behavior

- Stream event sequence: `meta`, `data`, `messages`, `end`
- Cancellation uses the existing run registry behavior (`ActiveRunRegistry`)

### Runtime persistence defaults

- `OPENCLAW_HOME=/mnt/openclaw`
- `OPENCLAW_STATE_DIR=/mnt/openclaw/state`
- `OPENCLAW_CONFIG_PATH=/mnt/openclaw/openclaw.json`
- `OPENCLAW_LOG_PATH=/mnt/openclaw/logs/openclaw-gateway.log`
- `OPENCLAW_PORT=18789`

Generated OpenClaw config enables:

- `gateway.http.endpoints.responses.enabled=true`

### Marketplace onboarding fields (v1)

1. `agent_name`
2. `agent_description`
3. `openclaw_model` (default `openclaw:main`)
4. `agent_instructions` (optional)
5. `openclaw_config_overrides_json` (optional)

### Manual FastAPI wiring

```python main.py
from agency import create_agency
from agency_swarm.integrations.fastapi import run_fastapi
from agency_swarm.integrations.openclaw import attach_openclaw_to_fastapi

app = run_fastapi(
    agencies={"openclaw": create_agency},
    return_app=True,
    app_token_env="APP_TOKEN",
)
if app is None:
    raise RuntimeError("FastAPI app failed to start")

attach_openclaw_to_fastapi(app)
```
